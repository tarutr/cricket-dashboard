# Round-6 items #11 + #12 — progress note

## Item #11 — player-popup "Vs opposition" table: cap + show-more/less

**Files:** `src/playerSections.js`, `styles.css`

- `oppositionSectionHTML()` (playerSections.js): when there are rows, wraps the
  existing `miniTableHTML` output in `<div class="vs-opposition">` containing a
  `.vs-opposition__scroll` (the table) + a `.vs-opposition__toggle` button
  (starts `hidden`). No-rows case is untouched (still the plain "No rows in
  this scope" note, no wrapper). The table's own computed rows are byte-identical
  — only the wrapper/toggle markup around it is new.
- New `wireOppositionToggles(root)` + a module-load-time
  `MutationObserver` on `#player-popup-host` (a static element already in
  index.html) find this markup wherever `battingGridHTML`/`bowlingGridHTML`
  land in the live popup DOM (open, reopen, discipline toggle, Filters apply,
  retry) and:
  - measure the LEFT `.player-page__col`'s rendered height (per player, per
    batting/bowling tab, since it's read live off the DOM each time)
  - if the opposition table's natural height already fits within that, hide
    the toggle entirely (nothing to collapse — no dead affordance)
  - otherwise clip the table to that height, show "Show more" (expand to
    `max-height:none`) / "Show less" (re-collapse)
  - each toggle button is marked `data-wired` once processed so re-firing the
    observer on unrelated popup mutations (e.g. typing in "find another
    player") is a cheap no-op; a genuine re-render replaces the DOM nodes
    wholesale so the new button always arrives unmarked.
- CSS: `.vs-opposition` / `.vs-opposition__scroll` / `.vs-opposition__toggle`
  (styles.css, near `.mini-table-wrap`). Also added `align-items: start` to
  `.player-page__two-col` — CSS Grid's default `stretch` was making both
  columns report the SAME (row-stretched) height, which would have broken the
  height match; `start` lets each column size to its own content (no visible
  difference before this fix — neither column paints a background/border that
  depended on the stretch).
- Also touched (not on the owned-files list, but necessary): the wiring
  approach was chosen specifically so `src/playerPage.js` (the actual DOM
  owner / re-render call site) did NOT need to change — see the doc comment
  on `wireOppositionToggles` for why. playerPage.js is untouched.

## Item #12 — "Show top 50" button on the Stats table

**Files:** `src/table.js`, `styles.css` (no CSS changes were actually needed —
reused `.table-show-more` / `.btn.btn--ghost` as-is)

- Added a second button `data-role="show-top50-btn"` inside the existing
  `.table-show-more` wrap in `ensureSkeleton()`'s skeleton markup (same
  wrapper/classes as "Show More", so identical placement/styling for free).
- New `showTop50BtnEl` ref (queried, reset to null in `teardownSkeleton()`
  alongside the other skeleton refs).
- Click handler sets `visibleRowCount = PAGE_SIZE` (the SAME local variable
  "Show More" already used, already local to the table controller — nothing
  added to `src/state.js`) then re-renders via the existing
  `renderLoaded(lastRows, lastLoadedState ?? store.get(), lastBowlingTypes)`
  path — pure re-render of already-loaded rows, no requery, matching Show
  More's own shape.
- `renderLoaded()`'s show-more block now also computes `isExpanded =
  visibleRowCount > PAGE_SIZE` and shows exactly one of the two buttons:
  "Show More (N players)" while there's more to reveal, "Show top 50" once
  expanded past the cap and nothing remains. The wrap itself only hides when
  NEITHER applies (result set never exceeded PAGE_SIZE in the first place).
